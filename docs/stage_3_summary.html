Stage 3 — ReAct Single Agent

Vòng lặp Think → Act → Observe tự động với create_react_agent

⭐⭐⭐ Nâng cao 25 phút LangGraph · prebuilt

1. Ý tưởng — ReAct là gì?

ReAct = Reasoning + Acting. Thay vì bạn viết tay 1 vòng tool-call (như Stage 2), agent tự lặp:

🧠THINK
Suy luận: cần gì?
→
⚡ACT
Gọi tool
→
👁️OBSERVE
Đọc kết quả
↻
✅ANSWER
Đủ thông tin?

Khác biệt cốt lõi với Stage 2:

  • Stage 2 — vòng lặp 1 lần, do code điều khiển
  • Stage 3 — vòng lặp N lần, do LLM tự quyết khi nào dừng

2. Code chính

File: stages/stage_3_single_agent/main.py

from langgraph.prebuilt import create_react_agent

llm = get_llm()
agent = create_react_agent(
    model=llm,
    tools=TOOLS,
    prompt=SYSTEM_PROMPT,
)

inputs = {"messages": [{"role": "user", "content": QUESTION}]}

# Cách 1: lấy kết quả cuối

3. Hai node graph mặc định

create_react_agent tạo ra một graph cực kỳ đơn giản:

┌──────────┐ │ START │ └────┬─────┘ ▼ ┌──────────┐ │ agent │ ← LLM, có thể tool_calls └────┬─────┘ │ ┌────┴─────┐ │ decide │ └─┬──────┬─┘ │ │ có tool│ │không tool ▼ ▼ ┌────────┐ ┌──────┐ │ tools │ │ END │ → final answer └────┬───┘ └──────┘ │ └─→ quay lại agent
NodeVai trò
agentGọi LLM. Trả về AIMessage có thể chứa tool_calls hoặc content (final answer)
toolsExecute mọi tool LLM yêu cầu (chạy song song trong cùng vòng). Trả ToolMessage

5. Bài tập đã hoàn thành

Exercise 3.1 — Thêm tool search_case_law

File: exercises/exercise_3_1_case_law.py

Agent có 3 tools: tra statutes, tra án lệ (case law), và calculate damages. Mỗi tool có domain riêng biệt.

Knowledge base án lệ mới

  • Hadley v. Baxendale (1854) — consequential damages
  • Donoghue v. Stevenson (1932) — duty of care
  • Carlill v. Carbolic Smoke Ball (1893) — unilateral contract
  • PepsiCo v. Redmond (1995) — inevitable disclosure doctrine
  • Winter v. NRDC (2008) — preliminary injunction standard

Kết quả thực nghiệm

Câu hỏi: "A former employee took confidential customer lists and joined a competitor. We had an NDA worth $200K. What statutes apply, what landmark cases support an injunction, and what damages can we expect?"

Agent gọi cả 3 tools song song trong cùng 1 step:

[Step 1] 🧠 THINK + ⚡ ACT
  🔧 search_legal_database({'query': 'trade secret misappropriation statutes'})
  🔧 search_case_law({'topic': 'injunction trade secret misappropriation'})
  🔧 calculate_damages({'breach_type': 'willful', 'contract_value': 200000})

[Step 2-4] 👁️  OBSERVE (3 kết quả từ 3 tools)

[Step 5] ✅ FINAL ANSWER  → cite DTSA + PepsiCo v. Redmond + Winter test
                          + damages $400K (2x willful multiplier)
💡 Đây là điểm khác biệt then chốt với Stage 2: agent đọc system prompt "cite BOTH statute AND case", tự lên kế hoạch gọi đủ 3 tools cùng lúc — Stage 2 sẽ chỉ gọi 1 tool/turn.

Exercise 3.2 — Debug agent reasoning

File: exercises/exercise_3_2_reasoning.py

Codelab gốc nói "thêm verbose=True" nhưng API mới không có param này. Thay vào đó, bài này so sánh 2 cách chạy agent:

Cách (A) — invoke() — silent

Output: "Số thuế thu nhập doanh nghiệp phải nộp là $200,000.00."
→ Bạn không biết agent đã làm gì để ra số này.

Cách (B) — astream() — full reasoning trace

┌─ [Step 1] 🧠 THINK + ⚡ ACT
│   Quyết định gọi: get_tax_rate
│   Với args:        {'country': 'Vietnam'}
└─

┌─ [Step 2] 👁️  OBSERVE
│   Kết quả từ tool: 20% (standard corporate income tax)
└─

┌─ [Step 3] 🧠 THINK + ⚡ ACT       ← agent tự parse "20%" → rate=20
│   Quyết định gọi: calculate_tax
│   Với args:        {'revenue': 1000000, 'rate_percent': 20}
└─

┌─ [Step 4] 👁️  OBSERVE
│   Kết quả từ tool: Tax on $1,000,000.00 at 20.0% = $200,000.00
└─

┌─ [Step 5] ✅ FINAL ANSWER
💡 Đây chính là multi-step planning mà Stage 2 không làm được: agent dùng output của tool 1 (string "20%") làm input cho tool 2 (số 20). Stage 2 thì user phải tự nối tay 2 lệnh.

Khi nào dùng cái nào?

ModeKhi nào dùng
ainvoke()Production — chỉ cần kết quả cuối
astream(stream_mode="updates")Debug — xem từng step ReAct
astream(stream_mode="messages")UI streaming — gửi từng token về frontend

  4. Sự khác biệt giữa stream_mode="updates" vs "values" vs "messages"?
