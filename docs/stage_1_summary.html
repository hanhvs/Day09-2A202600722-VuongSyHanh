1. Ý tưởng

LLM ở dạng cơ bản nhất là một API stateless: nhận text vào → trả text ra. Toàn bộ câu trả lời đến hoàn toàn từ training data của model.

User question
→
[SystemMessage + HumanMessage]
→
LLM
→
Answer

Stage 1 không có:

  • ❌ Tools / function calling
  • ❌ Tra cứu database (RAG)
  • ❌ Memory giữa các lần gọi
  • ❌ Multi-step reasoning tự động

2. Code chính

File: stages/stage_1_direct_llm/main.py

from langchain_core.messages import HumanMessage, SystemMessage
from common.llm import get_llm

llm = get_llm()

messages = [
    SystemMessage(content="You are a legal expert. Keep response under 300 words."),
    HumanMessage(content="What are the legal consequences if a company breaches an NDA?"),
]

response = await llm.ainvoke(messages)
print(response.content)

Vai trò 2 loại Message

MessageVai tròTương tự như
SystemMessageĐặt persona, ràng buộc style cho LLM"Job description" của LLM
HumanMessageCâu hỏi/yêu cầu thật của userInput từ user

💡 Đặc điểm thiết kế:
  • Lazy import — thiếu langchain-anthropic không gãy app nếu đang dùng OpenAI
  • Có error message rõ ràng khi thiếu key
  • Backward compat — default vẫn là openrouter

5. Bài tập đã hoàn thành

Exercise 1.1 — Đổi câu hỏi

File: exercises/exercise_1_1_question.py

Đổi QUESTION sang câu hỏi pháp lý tiếng Việt:

"Theo Bộ luật Lao động Việt Nam 2019, doanh nghiệp có quyền đơn phương chấm dứt hợp đồng lao động trong những trường hợp nào?"

Quan sát: LLM trả lời được 5 trường hợp dựa trên training data, có dẫn ra Điều 38 BLLĐ. Tuy nhiên không thể đảm bảo số điều/điều khoản luôn chính xác vì không tra cứu được luật thực tế.

Exercise 1.2 — Temperature control

File: exercises/exercise_1_2_temperature.py

Gọi LLM 4 lần với cùng câu hỏi NDA: 2 lần temperature=0.0, 2 lần temperature=1.0.

Kết quả thực nghiệm

LầnTemperatureSo sánh 2 lần liên tiếp
1 & 20.0✅ Giống hệt nhau (True)
3 & 41.0❌ Khác nhau (False)

Ở temperature=1.0, mỗi lần xuất hiện những điểm khác nhau (vd "Khiếu nại pháp lý", "Tổn hại đến uy tín") không có trong các lần 0.0.

Quy tắc chọn temperature

🧊 Thấp (0.0 – 0.3)

  • Phân tích pháp lý (cần chính xác, có thể audit)
  • Tóm tắt văn bản, trích xuất dữ liệu
  • Function calling / tool use

🔥 Cao (0.7 – 1.2)

  • Brainstorming, đề xuất ý tưởng
  • Viết quảng cáo, content sáng tạo
  • Sinh test cases đa dạng
